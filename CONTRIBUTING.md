# Contributing to Aegis-MCP

First off, thank you for considering contributing to Aegis-MCP! It's people like you that make Aegis-MCP such a great tool.

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues as you might find that the problem has already been reported. When you are creating a bug report, please include as many details as possible:

* **Use a clear and descriptive title** for the issue to identify the problem.
* **Describe the exact steps which reproduce the problem** in as many details as possible.
* **Explain which behavior you expected to see and why** and why you found the current behavior problematic.
* **Include screenshots** if applicable.

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please include:

* **Use a clear and descriptive title** for the issue to identify the suggestion.
* **Provide a step-by-step description of the suggested enhancement** in as many details as possible.
* **Explain why this enhancement would be useful** to most Aegis-MCP users.

### Pull Requests

* **Fill in the pull request template** (if available).
* **Include screenshots and animated GIFs** in your pull request whenever possible.
* **Follow the Go and React coding styles** used in the project.
* **Update the documentation** if you are adding or changing functionality.
* **Add tests** for your changes.

## Development Setup

See [GETTING_STARTED.md](docs/GETTING_STARTED.md) for instructions on how to set up your local development environment.

## Coding Standards

* **Go**: Follow standard Go idioms and use `go fmt`.
* **React**: Follow standard React practices and use `eslint`.
